About the Project


The current Department of Education, Skills and Employment (DESE) site has some information architecture (IA) and user interface (UI) design issues where users feel mentally exhausted going through an overwhelming amount of information in the navigation bar and web pages to navigate themselves to the desired site page. The solution was to redesign the navigation system and reorganise the presentation of existing information to reduce the cognitive load for users when interacting with the website.


Role & Responsibilities: As one of the main UI designers, I have been involved in all phases of the project. Main responsibilities were conducting user interface research, determining the information architecture of the website, illustrating design ideas and prototyping.


Duration: 5 weeks


Methods: User Interview, Persona Creation, Wireframes, Prototyping


Tools: Adobe XD, InVision, Miro, Usability Hub, Zoom

RESEARCH

User Interface Research outcomes


We have utilised three research methods to identify any UI pain points that users experience while interacting with the DESE website: heuristic evaluation, colour accessibility test and moderated remote usability tests.

Secondary navigation (Desktop)


The website adopted the conventional dropdown menus for its desktop navigation. However, the number of links to read through within the secondary navigation overwhelmed most users.

Menu (Mobile)


The website uses the hamburger menu for its mobile navigation. Again, the amount of information being presented within the menu was reported as overwhelming to most users.

Website cards


Too much negative space in the card. The information provided in the cards is pretty similar to the primary navigation bar. It is redundant to have both.

Search bar


Users expected the search bar would stay at its original spot when searching instead of directing users to a different search page.

Card Sorting


Cards for all the primary and secondary pages that appear in the DESE website navigation were created. We created a new website structure by using the define, group and structure approach.

ITERATE

User Interface Design Iterations

Usability Testing

The objective was to pinpoint any UI design issues with the high-fidelity mobile and desktop prototypes with functioning navigation through completing a series of tasks.


  • - 7 users, 5 tasks.
  • - 100% success rate for each task.
  • - More than 50% of participants struggled to drag the carousel on the mobile design.
  • - More than 40% of participants experienced different levels of difficulties tapping on the dropdown menu for the mobile design.
